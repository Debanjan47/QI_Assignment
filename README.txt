1. The five test cases are written in BDD methodology in Gherkin format in QA_Automation.feature feature file.
2. It's equivalent automation stepdefinition class is present in QA_Automation_stepdef.java file
3. Action class and locators are also present with names QA_Automatio_Action and QA_AutomationpageLocators java files
4. Use TestRunner java file for running the scripts
5. All 5 Test scripts are given a common tag "@QI" if needed to run all scripts together. Individual tag is also mentioned as @tag1, @tag2, .... respectively
